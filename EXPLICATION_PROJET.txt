================================================================================
         EXPLICATION DU PROJET RAG — Dictateurs & Autocraties
================================================================================

C'EST QUOI CE PROJET ?

  RAG = Retrieval-Augmented Generation.

  1. On cherche dans une base vectorielle (Qdrant) les morceaux de texte
     pertinents par rapport à la question.
  2. On les envoie au LLM comme "contexte" pour qu'il génère une réponse
     basée sur ces documents.

  Résultat : un chatbot spécialisé sur les dictateurs et régimes autoritaires
  qui répond à partir d'articles Wikipedia et de papers académiques.


STACK TECHNIQUE

  Composant         | Outil
  ------------------|----------------------------------------------
  Interface UI      | Chainlit (app.py)
  LLM               | llama-4-maverick via FAL.run
  Embeddings        | text-embedding-3-small via FAL.run (dim 1536)
  Base vectorielle  | Qdrant Cloud (collection "dictateurs")
  Orchestration     | LangChain (LCEL)
  Reranking         | BM25 via rank_bm25
  Sources           | Wikipedia FR + OpenAlex (papers)
  Déploiement       | Docker + GitHub Actions + Azure Web App

PIPELINE DE DONNÉES

  data/dictators.txt        (260 noms)
  data/related_topics.txt   (83 sujets)
          |
          v
  core/sources.py           télécharge les articles Wikipedia + OpenAlex
          |
          v
  assets/wikipedia/*.txt    (330 articles)
  assets/openalex/*.txt     (85 papers)
          |
          v
  core/ingestion.py         chunk (1000 chars) → embed (FAL API) → upsert Qdrant
          |
          v
  Qdrant Cloud              ~16 000 vecteurs


PIPELINE DE REQUÊTE (à chaque question)

  Utilisateur tape une question
          |
          v
  app.py : détection prompt injection (utils/guard.py)
          |
          v
  core/retrieve.py : recherche vectorielle dans Qdrant
    - similarity_search_with_relevance_scores (seuil 0.3)
    - 20 chunks candidats récupérés
    - rerank BM25 → 5 chunks finals
          |
          v
  core/generate.py : envoi du contexte + question au LLM
    - prompt système en français
    - streaming token par token vers Chainlit
          |
          v
  app.py : affichage réponse + sources


SIMILARITY SEARCH

  La question est convertie en vecteur (1536 dimensions).
  Qdrant calcule la similarité cosinus avec chaque chunk stocké.
  Les K chunks les plus proches sont retournés.

    similarité = cos(θ) = (A · B) / (||A|| × ||B||)
    1 = identique, 0 = sans rapport


MMR (Maximal Marginal Relevance)

  Équilibre pertinence et diversité.
  Au lieu de retourner les K plus proches (potentiellement redondants),
  MMR pénalise les chunks trop similaires à ceux déjà sélectionnés.

    MMR(chunk) = λ × sim(query, chunk) - (1-λ) × max(sim(chunk, déjà_retenu))


RERANK BM25

  Deuxième passe de scoring après la recherche vectorielle.
  BM25 cherche les correspondances exactes de mots-clés (complémentaire
  à la recherche sémantique).

  Pipeline : Qdrant → 20 chunks (vectoriel) → BM25 rerank → 5 chunks finals


INGESTION INCRÉMENTALE

  Chaque chunk a un ID déterministe (SHA-256 du contenu + source → UUID).
  Avant d'embedder, on vérifie dans Qdrant quels IDs existent déjà.
  Seuls les nouveaux chunks sont embeddés et envoyés → pas de coût inutile.


PROMPT INJECTION PROTECTION

  Couche 1 : regex sur l'input (patterns de vigil-llm + PayloadsAllTheThings)
  Couche 2 : canary token dans le system prompt, vérifié dans l'output


PARAMÈTRES CLÉS (config.py)

  RETRIEVAL_K       = 5           chunks envoyés au LLM (après rerank)
  RETRIEVAL_K_FETCH = 20          chunks récupérés depuis Qdrant
  RERANK_ENABLED    = True        activer/désactiver le rerank BM25
  RETRIEVAL_TYPE    = SIMILARITY  stratégie de search (SIMILARITY ou MMR)
  RELEVANCE_THRESHOLD = 0.3       score minimum de similarité
  chunk_size        = 1000        taille max d'un chunk en caractères
  chunk_overlap     = 200         chevauchement entre chunks consécutifs
  QDRANT_COLLECTION = "dictateurs"


COMMANDES

  uv run python -m core.sources                              télécharger les sources
  uv run python -m core.ingestion                            ingérer dans Qdrant
  uv run python -m core.ingestion delete assets/wikipedia/X.txt  supprimer une source
  uv run chainlit run app.py                                 lancer l'app
